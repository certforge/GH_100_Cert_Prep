# Lab Submissions

This directory is used by the **fork-only** GH-100 lab verification workflow.

## How It Works

1. Work in your **fork** of this repository.
2. Open a lab issue using the issue template.
3. Open a PR in your fork.
4. Add one markdown submission file under:
   - `lab-submissions/domain-1-enterprise-support/`
   - `lab-submissions/domain-2-identity-authentication/`
   - `lab-submissions/domain-3-deployment-licensing/`
   - `lab-submissions/domain-4-access-permissions/`
   - `lab-submissions/domain-5-security-compliance/`
   - `lab-submissions/domain-6-github-actions/`
5. Use the template in `lab-submissions/TEMPLATE.md`.

The workflow intentionally no-ops in the upstream source repository.
